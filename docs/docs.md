# 需求

## 1. 背景和概述

b站的关注没有分类功能，很难顶……

有的up我只是放在关注列表里，让算法能偶尔刷到他或者类似的视频，不代表每个我都想看。有的up我只想有大段时间的时候看。有的则是每天更新了就看。

所以要做一个app，能对up进行分类，然后展示出更新了、还没看的视频。

## 2. 页面总述

- 标签视图：展示不同标签下up主的视频
- up主管理：管理up主，管理up主的视频
    - 编辑up主的昵称、标签
    - 控制视频的已看、未看、不看
- 同步状态：手动触发同步，看上一次什么时候同步，同步情况

### 4.2. 全量同步

1. 按钮“立即同步”：点击后提交一个新的同步任务，提交后即可返回。
2. 界面展示进度条：轮询最近一次任务的完成情况，展示进度，展示结果
3. 展示最近一次同步记录

```jsonc
// SyncTask
{
    "id": "主键",
    "scope": "范围",
    "status": "状态",
    "started_at": "开始时间",
    "finished_at": "结束时间",
    "heartbeat_at": "最近心跳时间",
    "new_videos_count": "新视频的数量",
    "total_creators": "总共更新up数",
    "completed_creators": "更新成功up数",
    "current_creator_name": "当前更新up",
    "error_message": "失败信息，json，up和对应失败原因",
}
```

点击立即同步，创建这样一个对象。然后启动2个协程，1用来更新，2用来心跳（这样心跳其实跳的是应用有没有crash）
